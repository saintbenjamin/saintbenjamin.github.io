core.utils.bible_parser.resolve_book_name(name, lang_map=None, lang_code='ko')[source]

Resolve a user-provided book name (alias or standard) to the canonical internal ID.

This version supports: - Strict alias matching from BOOK_ALIASES - Reverse matching for canonical names - Normalized comparison (lowercase, remove spaces/dots) - Optional fallback to standard_book name mapping if provided

Parameters:

  • name (str) – Raw book name from user input (e.g., “요삼”, “1Jn”, “Genesis”)

  • lang_map (dict) – Optional standard_book.json dict (key → { “ko”: …, “en”: … })

  • lang_code (str) – Language key for matching standard book names (default: “ko”)

Returns:

Canonical book ID (e.g., “3John”), or None if not found

Return type:

str | None

core.utils.input_validators.validate_int(text, min_value=None, max_value=None)[source]

Validates whether the given input text is a valid integer, optionally enforcing minimum and/or maximum bounds.

Parameters:

  • text (str) – Input string to validate.

  • min_value (int, optional) – Lower bound (inclusive). Defaults to None.

  • max_value (int, optional) – Upper bound (inclusive). Defaults to None.

Returns:

  • is_valid (bool): True if the input is a valid integer and within bounds.

  • value (int or None): The parsed integer if valid; otherwise None.

Return type:

tuple

Example

>>> validate_int("42", min_value=10, max_value=100)
(True, 42)

>>> validate_int("abc")
(False, None)
